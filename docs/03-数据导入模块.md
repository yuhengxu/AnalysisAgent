# 03 · 数据导入模块

## 1. 职责

将外部文件（CSV / XLSX）转入平台：**落盘 → 识别类别 → 解析 → 入库 → 质量检查**。

- 服务：`backend/app/services/data_import.py`（`DataImportService`）
- 路由：`backend/app/api/data.py`
- 样例数据目录：`yuebao/`（价格表、供需平衡表、预测分析表）

## 2. 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/v1/data/upload?category=` | 上传单文件，category 可选（不传则自动识别） |
| POST | `/api/v1/data/seed` | 批量导入 `yuebao/` 下全部 `*.xlsx` 样例 |
| GET | `/api/v1/data/datasets` | 数据集列表 |
| GET | `/api/v1/data/quality/{dataset_id}` | 单数据集质量检查 |

## 3. 处理流程

```
upload(file, category?)
  └─ save_upload()                # 落盘到 data/raw/{时间戳}_{原名}
  └─ import_file(path, category)
        ├─ detect_category()      # 未指定时按文件名关键字猜测
        ├─ .csv  → _import_csv()   # 仅登记元数据（不逐行入库）
        └─ .xlsx → _import_xlsx()  # 按类别分流
              ├─ price   → _import_price_xlsx()    → price_series
              ├─ balance → _import_balance_xlsx()  → balance_forecasts
              ├─ factor  → _import_factor_xlsx()   → factor_assessments
              └─ generic → 仅登记 dataset
  └─ quality_check(dataset_id)    # 返回 issues / passed
```

## 4. 类别识别（detect_category）

按**文件名关键字**：
- 含「价格 / price」→ `price`
- 含「供需 / balance」→ `balance`
- 含「预测 / factor」→ `factor`
- 否则 `generic`

## 5. 三类 XLSX 解析要点

### 5.1 价格表 → `price_series`
- 表头列名经 `PRICE_SYMBOL_MAP`（`templates/monthly_report.py`）映射为标准 symbol（如「布伦特(期)」→ `Brent`，「ESPO」→ `ESPO`）。
- 支持 `原油价格表. 20260603.xlsx` 这类**多 Sheet 月度价格表**：每个 sheet 第一行可横向列出品种，日期行与汇总行混排。
- 自动探测表头位置，不要求从 A1 开始；同一日期同一 symbol 重复列只保留首个有效值。
- 逐行解析日期（`_parse_date` 支持 `YYYYMMDD` / `YYYY-MM-DD` / Excel 日期对象）。
- 跳过含「平均 / 本周 / 较上月 / 变化率 / 环比 / 同比」的汇总行。
- 代码级 upsert：按 `symbol + trade_date + source` 更新已有数据，不插入重复业务行。

**下游消费（只读 DB，不再读 Excel）：**

| 消费方 | 用途 |
|--------|------|
| `calc_monthly_stats` | 月报表1-1 月均价、环比、同比、最高/最低、峰谷价差 |
| `query_price_series` / `chart_config` | 首页图表、月报图1-1/1-4/1-5/1-6 |
| `ForecastService` | 统计预测取历史序列 |

月报生成时 Excel **仅在本导入阶段使用**；导出图表与系统表均依赖预置 snapshot（表1-1 由 `price_series` 同步）。详见 [04-数据处理与分析模块](./04-数据处理与分析模块.md) §5、[16-月报表数据模块](./16-月报表数据模块.md)。

### 5.2 供需平衡表 → `balance_forecasts`
- 支持 `供需平衡表.xlsx` 的**多 Sheet + 偏移表头**：表头可从 B2、C3 等位置开始。
- 自动识别「预测机构 / 更新日期 / 供需」列，以及 `2026Q1`、`2026年`、`2026Q1供需差` 等周期列。
- 对机构/更新日期使用向下继承，兼容“供/需”两行成组的数据布局。
- 代码级 upsert：按 `agency + update_date + supply_demand + period` 更新已有数据，不插入重复业务行。

### 5.3 预测因素表 → `factor_assessments`
- 从文件名提取 `report_month`（`YYYY年M月` → `YYYY-MM`）。
- 识别大类行（`1.宏观…`）与因素行（`1.1 …`）。
- ⚠️ **已知缺陷**：importance 用 `read_only+data_only` 读取，丢失"下划线选中"格式，当前取行内最后一个 1-5 值 ⇒ importance 常恒为 5，impact 取行内匹配项。详见 14-TODO-IMPORT-1。

## 6. 质量检查（quality_check）

当前仅对 `price` 类别校验：
- 是否有价格行入库；
- 是否识别到 ≥2 个品种。

其余类别返回空 issues（视为通过）。增强见 14-TODO-IMPORT-2。

## 7. 关键文件
- `services/data_import.py`：全部解析逻辑。
- `templates/monthly_report.py::PRICE_SYMBOL_MAP`：品种名映射。
- `api/data.py`：路由 + 样例目录定位（兼容 Docker `/app/yuebao` 与本地 `../yuebao`）。
