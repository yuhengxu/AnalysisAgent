## 2. 结构化内容模型（content_json）
```jsonc
{
  "cover":   { "org": "...", "title": "国际油价月报", "issue": "...", "dept": "...", "date": "..." },
  "summary": "内容摘要，可多段（\\n 分隔）",
  "sections": [
    { "id": "review", "title": "一、原油市场回顾", "level": 1 },
    { "id": "review_futures", "title": "（一）期货市场回顾", "level": 2, "content": "正文…", "hint": "提示" },
    // … 共 17 个章节节点：3 大节(level1) + 14 子节(level2)
  ],
  "tables": {
    "table_price_change": { "title": "...", "source": "CNEEI", "headers": [...], "rows": [[...]] },
    "table_macro_pmi":    { ... },
    "table_demand_forecast": { ... },
    "table_supply_balance": { ... },
    "table_scenario":     { ... },
    "table_agency":       { ... },
    "table_distribution": { ... }
  },
  "approval": { "author": "执笔：…", "reviewer": "初审：…", "approver": "审核：…", "signer": "签发：…" }
}
```
大纲固定在 `templates/monthly_report.py::DEFAULT_SECTIONS`，表格在 `DEFAULT_TABLES`。

- 内容摘要
- 一、原油市场回顾：(一)期货市场回顾[图1-1/图1-2/表1-1] / (二)现货市场回顾[图1-4/图1-5/图1-6]
- 二、国际油价影响因素：(一)宏观金融[表2-1] (二)需求[表2-2] (三)供应[图2-1/表2-3] (四)库存 (五)美元美联储 (六)地缘 (七)持仓情绪
- 三、国际油价展望：(一)情景分析[表3-1] (二)研讨会 (三)模型 (四)机构预测[表3-2] (五)预测结论

## 4. 回顾月与报告期

| 概念 | 说明 | 示例（6 月月报） |
|------|------|------------------|
| **报告期** | 封面期号、报告日期 | 2026 年第 6 期，日期 2026 年 6 月 |
| **回顾月** | `review_year` + `review_month`，数据与正文回顾口径 | 2026 年 **5** 月 |
| **展望月** | `outlook_year` + `outlook_month`，第三章预测口径 | 2026 年 **6** 月 |

生成请求体必须显式传入回顾月；表1-1、图1-1～1-6 的数据区间均为**回顾月首末日**，存于 `evidence_json.review_year/review_month`。

## 6. 生成流程
```
ReportService.generate_monthly_draft(issue_no, report_date, review_month, outlook_month, ...)
  └─ ensure_default_template()
  └─ ReportSkill.generate() 或 ReportUnrestrictedSkill.generate()
       ├─ gather_evidence(review_year, review_month)   # Brent/WTI + spot_market（图1-4~1-6 同源）
       ├─ if llm.is_enabled():  _generate_with_llm() → _merge()   # 不合并 table_price_change
       │  else:                  _fallback()
       ├─ guard_report_content()
       └─ ReportTableDataService.load_for_report()   # ★ 6 张系统表从 snapshot 加载
  └─ 落库 reports(draft) → chart_export 生成 PNG → export_docx() → 回写 docx_path
```

- `_merge()`：覆盖 summary/各 section.content/approval；**跳过**全部 6 张 `SYSTEM_TABLE_KEYS`。
- 兼容：旧扁平 content 在导出时经 `_upgrade_legacy()` 自动升级为结构化。

## 7. REST 接口
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/reports/generate` | 同步生成 |
| POST | `/reports/generate/async` | 异步生成（切换页面不中断） |
| GET | `/reports/generate/tasks/{task_id}` | 轮询进度（**校验 task.user_id**） |
| GET | `/reports` | 列表（**仅当前用户**） |
| GET | `/reports/template` | 默认模板结构 |
| GET | `/reports/{id}` | 详情（**仅当前用户**） |
| PUT | `/reports/{id}` | 保存校准（content/title），status→reviewed |
| GET | `/reports/{id}/export` | **导出 Word/PDF/tex**（仅当前用户） |
| GET | `/reports/{id}/charts/{chart_id}` | **章节图表 PNG**（需 JWT + 归属校验） |
| DELETE | `/reports/{id}` | 删除（连带删 docx） |
| POST | `/agent/revise/{id}` | **章节级校准**（归属校验，越权 404） |

### 7.1 用户隔离

`ReportService._scope_query` / `get_report` 严格按 `user_id == 当前用户.id` 过滤，**管理员也不能查看他人月报**。历史 `user_id IS NULL` 记录在启动时归属默认 admin。详见 [17-用户管理与权限](./17-用户管理与权限.md)。

## 8. 导出 Word
`ReportService.export_docx()`（python-docx）：封面 → 摘要 → 逐章节 → 按 `REPORT_CHART_ANCHORS` 插入 PNG → 按 `REPORT_TABLE_ANCHORS` 插入表格 → 审核签发。输出 `data/exports/report_{id}_{时间戳}.docx`。

图表 PNG 输出到 `data/charts/`；无效小文件（<1KB 占位图）会在打开报告时自动重新生成。

## 9. 章节级 Agent 校准
`agent.revise_selection(report_id, section_id, instruction, ...)`：
- 取该 section 原文 → `revise_context.build_revise_context()` 组装上下文 → `_llm_revise()` 改写 → 回写 content。
- **不修订** 6 张系统表 rows（由数据中心 snapshot 维护；打开/导出时从 snapshot 重载）。

## 10. 前端图表预览
ReportCenter 通过 `ReportChartImage` 组件带 Token 加载 `GET /reports/{id}/charts/{chart_id}`，与 Word 导出一致。
