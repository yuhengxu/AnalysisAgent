# 09 · 数据流转与时序

## 1. 数据生命周期总览

```
外部文件(xlsx/csv) / 大模型知识 / 可信数据源
        │ 导入
        ▼
   data/raw/ (原始落盘)  ──解析──►  SQLite 表
        │                          (price_series / balance_forecasts / factor_assessments)
        │ 读取
        ▼
   AnalyticsService (统计/价差/图表)
        │ 作为"证据"
        ▼
   Skill (prediction / report) ──调用──► 大模型（豆包/DeepSeek）
        │ 结构化结果 content_json
        ▼
   predictions / reports 表  ──编辑/校准──►  前端
        │ 导出
        ▼
   data/exports/*.xlsx / *.docx  ──下载──► 用户
```

## 2. 存储位置一览
| 数据 | 位置 |
|------|------|
| 原始上传文件 | `data/raw/{时间戳}_{原名}` |
| 结构化业务数据 | SQLite `db/energy_platform.db` |
| 导出文件 | `data/exports/*.xlsx` `*.docx` |
| 运行日志 | `logs/app.log`（滚动） |
| 样例数据 | `yuebao/`（只读参考） |

## 3. 关键链路时序

### 3.1 导入价格数据
```
前端 DataCenter ──POST /data/upload(file,price)──► api/data
  api/data → DataImportService.save_upload() → data/raw
           → import_file() → _import_price_xlsx() → INSERT price_series
           → quality_check()
  ◄── {import, quality}
前端刷新 /data/datasets，并拉 /analytics/charts/price_trend 预览
```

### 3.2 生成预测分析表（核心）
```
前端 Prediction ──POST /prediction/generate{year,month,model_*}──► api/prediction
  → PredictionService.generate()
      → PredictionSkill.generate()
          → gather_evidence()                  [读 analytics → DB]
          → llm.chat_json(SYSTEM, user含可信源+证据)   [→ 大模型]
          → _normalize() 校正 28 项
      → INSERT predictions(draft)
  ◄── {id, content, sources, llm_used}
前端渲染可编辑表格 → 人工校准 → PUT /prediction/{id}(content)
                  → GET /prediction/{id}/export → _build_xlsx() → 下载 Excel
```

### 3.3 生成月报（核心）
```
前端 ReportCenter ──POST /reports/generate{review_year, review_month, outlook_*, ...}──► api/reports
  → ReportService.generate_monthly_draft()
      → ReportSkill.generate()
          → gather_evidence(review_year, review_month)
               → calc_monthly_stats(Brent/WTI)  [读 price_series]
          → llm.chat_json → _merge()           [正文；跳过 6 张 SYSTEM_TABLE_KEYS]
          → ReportTableDataService.load_for_report()  [★ 6 张系统表从 snapshot 加载]
      → INSERT reports(draft)
      → chart_export.generate_report_charts()  [★ 图1-1～1-6 只读 DB，区间=回顾月]
      → export_docx()                          [导出前 _refresh_report_tables()]
  ◄── {id, content, review_year, review_month, ...}

前端打开 GET /reports/{id}
  → get_report_detail() → _refresh_report_tables()  [系统表与 snapshot 同步]

前端编辑/Agent 修订 → PUT /reports/{id} → GET /reports/{id}/export → 下载 Word
```

**回顾月示例：** 生成「2026 年第 6 期」月报时传 `review_year=2026, review_month=5`，表1-1 展示 5 月 Brent/WTI 月均价及环比/同比，而非 6 月或 LLM 臆造值。

### 3.4 对话式 Agent
```
前端 AgentChat ──POST /agent/chat{messages}──► agent.chat
  → _detect_visual_intent? → 调外观，返回 visual
  → _detect_skill → run(skill) → (prediction/report/forecast/analytics)
  → 落 agent_runs
  ◄── {message, visual, charts, report_id?, prediction_id?}
前端据 *_id 提供"打开预测分析表/报告中心"跳转
```

## 4. 事务与一致性
- 每请求一个 Session（`get_db`），服务内部 `commit()` 后 `refresh()`。
- 文件产出与 DB 路径回写：先存库拿到 id → 生成文件 → 回写 `xlsx_path/docx_path` 再 commit。
- 当前为单机单库，无分布式事务需求。
