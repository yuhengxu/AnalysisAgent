# 12 · 系统启动与部署

## 1. 本地开发

### 后端
```bash
cd backend
python -m venv .venv && source .venv/bin/activate    # 已存在则跳过
pip install -r requirements.txt
# 配置大模型
cp .env.example .env   # 填入 VOLCENGINE_API_KEY（或 DEEPSEEK_API_KEY）
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```
- 健康检查：http://localhost:8000/health
- 交互式 API 文档：http://localhost:8000/docs （FastAPI 自带 Swagger）

### 前端
```bash
cd frontend
npm install
npm run dev      # http://localhost:5173 （已配 /api 代理到 8000）
```
首次使用：
1. 进入「数据中心」→「导入」上传 yuebao 样例（或调用 `POST /data/seed`）
2. 进入「月报表数据」Tab → 选择回顾月 → **同步派生数据** → **联网获取**（可选）
3. 再前往「月报中心」生成月报

### 便捷脚本（前台单服务）
`scripts/start_backend.sh|bat`、`scripts/start_frontend.sh|bat`（阻塞式，适合开发时单独看某个服务日志）。

### 统一管理脚本（推荐 · 后台运行 + 启停重启）
`scripts/manage.sh`（Linux/macOS）与 `scripts/manage.bat`（Windows）。
```bash
./scripts/manage.sh start            # 启动前后端（后台）
./scripts/manage.sh start backend    # 只启动后端
./scripts/manage.sh stop [all|backend|frontend]
./scripts/manage.sh restart [目标]
./scripts/manage.sh status           # 查看运行状态/端口/PID
./scripts/manage.sh logs backend     # 跟踪日志(Ctrl+C 退出)
```
- 后台运行：PID 写入 `logs/{backend,frontend}.pid`，运行日志 `logs/{backend,frontend}.run.log`。
- 以独立进程组启动，停止时按组清理（正确回收 uvicorn `--reload` / vite 的子进程），并按端口兜底。
- 首次启动若缺 `.venv` / `node_modules` 会自动安装（`AUTO_INSTALL=0` 可关闭）。
- 可用环境变量覆盖：`BACKEND_PORT`(8000) `FRONTEND_PORT`(5173) `HOST`(0.0.0.0) `RELOAD`(1)。
  例：`RELOAD=0 BACKEND_PORT=8080 ./scripts/manage.sh start backend`。

## 2. Docker Compose
```bash
# 在宿主机导出 key（或写 .env，compose 已支持变量注入）
export DEEPSEEK_API_KEY=sk-...
docker compose up --build
```
- 前端 http://localhost:3000 ｜ 后端 http://localhost:8000
- 卷挂载：`./data ./db ./logs`（持久化）、`./yuebao:ro`（样例只读）。

## 3. 启动顺序（backend/app/main.py）
```
1. setup_logging(settings.logs_dir)     # 最先：初始化日志
2. create_app(): CORS + 请求日志中间件 + 注册 6 个路由
3. @startup: init_db() 建表 + ensure_default_template() 写默认月报模板
4. 暴露 /health
```

## 6. 月报图表引擎

月报 PNG（图1-1～1-6）支持双引擎，通过 `.env` 切换：

```env
CHART_RENDERER=matplotlib   # 默认
# CHART_RENDERER=echarts    # PyEcharts + Playwright
CHART_ECHARTS_FALLBACK=false  # 仅显式设为 true 时，echarts 失败才回退 matplotlib
CHART_ECHARTS_WIDTH=1400
CHART_ECHARTS_HEIGHT=780
CHART_ECHARTS_DEVICE_SCALE=3  # 导出像素比，PNG 约 4200×2340
```

**echarts 模式**需安装浏览器：

```bash
cd backend
pip install -r requirements.txt
playwright install chromium
```

### 月报 LaTeX 导出（PDF / Word）

PDF 与 LaTeX 管线 Word 导出依赖系统工具（非 pip 包）：

```bash
# Debian/Ubuntu
sudo apt install texlive-xetex texlive-lang-chinese pandoc
```

- **PDF**：`GET /reports/{id}/export?format=pdf`（xelatex 编译）
- **Word**：`GET /reports/{id}/export?format=docx`（优先 pandoc 转 LaTeX，不可用时回退 python-docx）
- **LaTeX 源码**：`GET /reports/{id}/export?format=tex`（可手工微调排版后自行编译）
- 模板文件：`backend/app/templates/latex/monthly_report_preamble.tex`

Docker 镜像已包含 `playwright install --with-deps chromium`。仅当 `CHART_ECHARTS_FALLBACK=true` 时，echarts 渲染失败才会自动回退 matplotlib。

## 7. 常见问题
- 大模型不生效：检查 `.env` 的 `VOLCENGINE_API_KEY`/`DEEPSEEK_API_KEY` 与 `DEFAULT_LLM_PROVIDER`；不可用时系统自动走规则兜底（仍可产出，但内容较简）。
- 豆包深度研究模式无效：检查 `VOLCENGINE_DEEPSEARCH_BOT_ID` 是否已配置（未配置时自动降级普通模式并写告警日志）。
- 端口冲突：改 uvicorn `--port` 与前端 `vite.config.ts` 代理 target。
- 样例导入找不到目录：确认 `yuebao/` 在项目根（Docker 下为 `/app/yuebao`）。
