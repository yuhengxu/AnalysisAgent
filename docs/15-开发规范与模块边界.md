## 2. 分层职责（不要越界）

| 层 | 能做 | 不能做 |
|----|------|--------|
| `api/` | 解析请求、调用服务、转换异常→HTTP | 写业务逻辑、直接访问 DB 模型 |
| `services/` | DB 读写、聚合、文件产出、编排 skill | 直接拼 HTTP 响应、解析 Request |
| `skills/` | 采集证据 + 调 LLM + 结构化产出 + 降级 | 直接处理 HTTP；耦合前端 |
| `core/` | 配置、DB 连接、LLM、日志（基础设施） | 业务逻辑 |
| `models/` | 表结构定义 | 业务方法 |
| `templates/` | 结构/常量定义 | 副作用逻辑 |

## 3. 新增一个 Agent skill 的标准步骤
1. `skills/xxx_skill.py`：写类，提供 `gather_evidence()`、`generate()`（含 `if llm.is_enabled(): ... else: _fallback()`）。
2. 如需持久化：加 `models/xxx.py` + `services/xxx.py`（CRUD + 导出）+ `api/xxx.py`，并在 `main.py` 注册路由、`models/__init__.py` 注册表。
3. 在 `services/agent.py`：`SKILLS` 加条目；`run()` 加 `elif skill=="xxx"` 分支；`_detect_skill()` 加关键词。
4. 前端：`api/client.ts` 加函数；必要时加页面/导航。
5. 文档：在本目录补对应说明，并在 `14-待完善功能清单` 勾掉相关项。

## 4. 编码约定
- **大模型必降级**：任何 LLM 调用都要 `try/except llm.LLMUnavailable` 并提供规则兜底，保证接口不因大模型失败而 500。
- **结构化内容存 JSON**：预测表/月报内容存 `content_json`，前端编辑与后端导出共用同一结构，**改结构要三处同步**（模板、导出、前端类型）。
- **系统表 snapshot**：6 张月报系统表（表1-1、2-1、2-2、2-3、3-1、3-2）由数据中心预置写入 `report_table_snapshots`，生成/打开/导出时只读加载，**不得依赖 LLM 填数或在生成流程实时计算**。
- **DB 会话**：只通过 `Depends(get_db)` 获取；服务内 `commit()` 后按需 `refresh()`。
- **错误风格**：服务层抛 `ValueError("X not found")`，路由层转 `HTTPException(404)`。
- **中文注释**：模块级 docstring 说明"职责/调用关系"，关键函数说明"输入/输出/副作用"。
- **不提交密钥**：`.env` 已被 `.gitignore` 忽略；新增密钥走环境变量。

## 5. 改动后自检清单
- [ ] 后端：`cd backend && python -c "from app.main import app"` 无报错
- [ ] 后端：相关接口用 curl/Swagger(/docs) 验证
- [ ] 前端：`cd frontend && npm run build` 通过（`tsc` 类型检查包含其中）
- [ ] 文档：涉及结构变化时同步更新 `docs/`
- [ ] 日志：关键路径有 `logger.info/warning`
