# 13 · 日志与可观测性

## 1. 日志模块（core/logging.py）
- `setup_logging(logs_dir, level)`：幂等初始化根日志器；**在 main.py 最先调用**。
- 双输出：控制台 + 滚动文件 `logs/app.log`（单文件 5MB，保留 5 份）。
- 格式：`时间 | 级别 | logger名 | 消息`。
- 级别：环境变量 `LOG_LEVEL`（默认 INFO）。
- 降噪：httpx/httpcore 降到 WARNING。

## 2. logger 命名约定
| logger 名 | 来源 | 内容 |
|-----------|------|------|
| `api` | main.py 中间件/启动 | 每个 HTTP 请求(方法/路径/状态/耗时)、启动事件 |
| `llm` | core/llm.py | 大模型调用失败、provider/model |
| `skill.prediction` | prediction_skill | 预测 skill 降级告警 |
| `skill.report` | report_skill | 月报 skill 降级告警 |
| `app` | logging.py | 日志系统自身 |

业务代码统一 `logging.getLogger("<语义名>")` 获取，无需配置 handler。

## 3. 请求日志中间件（main.py）
对每个请求记录：`{METHOD} {path} -> {status} ({ms} ms)`。可用于：
- 排查慢接口（大模型生成接口通常 10~30s）。
- 审计调用量。

## 4. 业务审计：agent_runs 表
每次 `/agent/run`、`/agent/chat` 落库（skill/prompt/provider/model/tools/evidence/response/charts/耗时），是 Agent 行为的可追溯记录（见 02）。

## 5. 排障速查
| 现象 | 看哪里 |
|------|--------|
| 接口 500 | `logs/app.log` 中该请求行附近的异常栈 |
| 大模型没生效 | 搜 `llm` logger 的 "LLM 调用失败" + 检查 key |
| skill 走了兜底 | 搜 `skill.prediction/report` 的 "大模型不可用，降级" |
| 生成很慢 | 看请求中间件耗时，确认是 LLM 往返 |
