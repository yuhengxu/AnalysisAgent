# 16 · 月报表数据模块

## 1. 功能定位

数据中心 **「月报表数据」** Tab 按**回顾月**预置月报 6 张动态表，生成月报时从 `report_table_snapshots` 只读加载，不再在生成流程中实时联网计算。

| 类别 | 说明 | 手工修正 |
|------|------|----------|
| `derived` | 从导入数据直接获取或计算 | 否（可重新同步） |
| `web` | 从 DeepSearch 联网获取 | 是（保存后锁定，联网不再覆盖） |

`table_distribution`（报送名单）为静态模板，不纳入本模块。

## 2. 六张表清单

| table_key | 表号 | 类别 | 数据源 | 期别 |
|-----------|------|------|--------|------|
| `table_price_change` | 表1-1 | derived | `price_series` → `calc_monthly_stats` | 回顾月 |
| `table_supply_balance` | 表2-3 | derived | `balance_forecasts`（供需差） | 回顾月 snapshot；无则回退至更早最新月 |
| `table_scenario` | 表3-1 | derived | `forecast_results` | 展望月 |
| `table_macro_pmi` | 表2-1 | web | S&P Global / Eurostat / 国家统计局 | 回顾月 |
| `table_demand_forecast` | 表2-2 | web | 手工填写（可选大模型预测） | 回顾月 |
| `table_agency` | 表3-2 | web | 手工录入 / 机构预测 | 回顾月存储（对应展望月 +1） |

存储主键：`(review_year, review_month, table_key)`。

## 4. 操作流程

1. 导入 yuebao 样例（价格、供需平衡表）— **价格/供需导入成功后自动同步对应回顾月的派生表**（表1-1 或表2-3）
2. 数据中心 → 月报表数据 → 选择**回顾月**
3. 点击 **同步派生数据**（表1-1、2-3、3-1；含表3-1 需有预测模型）
4. 点击 **深度研究联网获取**（表2-1 PMI 联网查证；表2-2 **默认手工填写**，可在界面勾选「大模型预测表2-2」后启用）
5. 生成月报 → 若系统表未预置，前端会提示确认；生成后 `load_for_report()` 注入 `content.tables`

**表3-2 期别**：数据中心按**回顾月**存储；兼容 API `/data/agency-forecasts` 仍接受展望月，服务层自动映射。

## 5. API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/data/report-tables/schema` | 6 表模板 |
| GET | `/data/report-tables?review_year=&review_month=` | 该月全部 snapshot |
| GET | `/data/report-tables/list` | 已录入月份列表 |
| POST | `/data/report-tables/sync-derived` | 同步派生表 |
| POST | `/data/report-tables/fetch-web` | 联网获取 web 表 |
| PUT | `/data/report-tables/{table_key}` | 手工保存 web 表 |

兼容：`/data/agency-forecasts` 仍可用，内部读写 `table_agency` snapshot（展望月 ↔ 回顾月自动映射）。

## 6. 月报衔接

```
ReportSkill.generate()
  → LLM 撰写 sections（不填 6 张系统表 rows）
  → ReportTableDataService.load_for_report(review, outlook)
  → 覆盖 content.tables 中 6 张表
```

打开详情 / 导出 Word：`ReportService._refresh_report_tables()` 同样从 snapshot 重载。

缺失表：返回空壳模板，`evidence.table_snapshots.missing_tables` 列出未预置项。

## 7. 数据模型 `report_table_snapshots`

- `review_year`, `review_month`, `table_key`（唯一）
- `source_category`: `derived` | `web`
- `title`, `source`, `headers_json`, `rows_json`
- `source_urls_json`, `is_manual_override`
- `computed_at`, `updated_by`, `updated_at`

启动时 `init_db()` 自动将旧 `agency_forecast_manual` 迁移至 snapshot（若尚未存在）。
