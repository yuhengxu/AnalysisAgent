# 17 · 用户管理与权限

> 本文档描述平台的多用户登录、页面权限、月报/预测数据隔离与异步任务归属校验。
> 实现方案详见项目根目录 `plans/edituser-plan.md`。

## 1. 能力概览

| 能力 | 说明 |
|------|------|
| 角色 | `admin`（管理员）、`user`（普通用户） |
| 用户管理 | 管理员可新增/删除普通用户、分配页面权限、启停账号、重置密码 |
| 初始密码 | 新建或重置的普通用户密码固定为 **`qwer1234`**（bcrypt 哈希入库） |
| 页面权限 | 普通用户仅可访问被授权的页面；前后端双重校验 |
| 数据隔离 | 月报、预测分析表严格按 `user_id` 隔离（**含 admin**） |
| 任务隔离 | 月报/预测异步生成任务绑定 `user_id`，轮询时校验归属 |
| 并发 | SQLite WAL + busy_timeout；异步任务为**单进程内存态** |

## 2. 页面 key 与导航

| key | 导航名称 | 主要 API 前缀 |

### 2.2 管理员专属页面（不分配给普通用户）

| key | 导航名称 | 权限 |
|-----|----------|------|
| `users` | 用户管理 | `require_admin` |

代码常量见 `backend/app/models/user.py`：`ALL_PAGE_KEYS`、`ADMIN_PAGE_KEYS`、`PAGE_LABELS`。

## 3. 数据库（users 表）

| 字段 | 说明 |
|------|------|
| id | 主键 |
| username | 用户名（唯一） |
| hashed_password | bcrypt 哈希 |
| role | `admin` / `user` |
| **allowed_pages_json** | 普通用户可访问页面 key 的 JSON 数组，如 `["dashboard","reports"]` |
| **is_active** | 是否启用；`false` 时登录与 API 均拒绝 |
| created_at | 创建时间 |
| updated_at | 最后更新时间 |

迁移逻辑在 `core/migrate.py`：启动时自动补列，并为历史普通用户填充默认全业务页面权限。

### 3.1 月报/预测隔离字段

| 表 | 字段 | 规则 |
|----|------|------|
| `reports` | `user_id` | 创建者 ID；列表/详情/导出/图表/修订均校验 `user_id == 当前用户.id` |
| `predictions` | `user_id` | 同上 |

**严格隔离**：管理员也只能看到自己的月报与预测表，不能查看其他用户数据。越权访问返回 **404**（避免泄露资源是否存在）。

历史 `user_id IS NULL` 的月报在启动时由 `services/user_seed.py` 归属默认 admin。

## 4. 后端鉴权

### 4.1 依赖函数（`core/deps.py`）

| 函数 | 用途 |
|------|------|
| `get_current_user` | 解析 JWT，校验用户存在且 `is_active` |
| `require_admin` | 要求 `role == admin` |
| `require_page(page_key)` | 管理员放行；普通用户须在 `allowed_pages()` 中 |

### 4.2 路由权限一览

| 模块 | 权限依赖 |
|------|----------|
| `/auth/*` | 登录公开；`/me` 需登录 |
| `/users/*` | `require_admin` |
| `/data/*` | `require_page("data")` |
| `/analysis/*` | `require_page("analysis")` |
| `/forecast/*` | `require_page("forecast")` |
| `/prediction/*` | `require_page("prediction")` |
| `/reports/*` | `require_page("reports")` |
| `/analytics/dashboard` | `require_page("dashboard")` |
| `/analytics/*`（其余） | 登录即可（多页面复用图表接口） |
| `/llm`、`/llm-logs` | `require_admin` |

JWT payload 仅含 `sub`（用户 ID）与 `role`。**页面权限不写入 JWT**，管理员修改权限后，用户刷新或调用 `/auth/me` 即可生效。

### 4.3 用户管理 API

基址：`/api/v1/users`（均需 admin + Bearer token）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/users/page-options` | 可勾选的业务页面列表 |
| GET | `/users` | 用户列表 |
| POST | `/users` | 创建普通用户（初始密码 `qwer1234`） |
| PUT | `/users/{id}` | 修改页面权限 / 启停（不可改 admin） |
| DELETE | `/users/{id}` | 删除普通用户（不可删自己或 admin） |
| POST | `/users/{id}/reset-password` | 重置为 `qwer1234` |

### 4.4 登录响应结构

```json
{
  "access_token": "...",
  "token_type": "bearer",
  "user": {
    "id": 1,
    "username": "zhangsan",
    "role": "user",
    "allowed_pages": ["dashboard", "reports"]
  }
}
```

## 6. 异步任务与并发

### 6.1 任务归属

月报与预测异步任务在创建时写入 `user_id`。查询 `GET .../generate/tasks/{task_id}` 时校验：

```python
if task.user_id != user.id:
    raise HTTPException(404, "任务不存在或已过期")
```

任务状态保存在进程内存字典（`report_tasks.py` / `prediction_tasks.py`），**不支持多 worker**。

### 6.2 生产部署约束

```bash
# 正确：单 worker
uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 1

# 错误：多 worker 会导致异步任务轮询 404
uvicorn app.main:app --workers 4
```

如需多 worker，须先将任务状态迁移到数据库或 Redis（见 `14-待完善功能清单` 后续增强项）。

### 6.3 SQLite 并发优化

`core/database.py` 已配置：

- `PRAGMA journal_mode=WAL`
- `PRAGMA busy_timeout=30000`
- `connect_args timeout=30`
- `pool_pre_ping=True`

## 7. 管理员操作指南

1. 使用 `.env` 中 `ADMIN_USERNAME` / `ADMIN_PASSWORD` 登录（首次启动自动创建 admin）。
2. 进入导航 **用户管理**（`/users`）。
3. 输入用户名，勾选可访问页面，点击 **创建用户**。
4. 将初始密码 `qwer1234` 告知新用户（建议首次登录后自行修改——当前版本尚未实现改密功能）。
5. 可随时调整权限、停用账号、重置密码或删除用户。

**限制**：

- 不能删除或修改 admin 账号权限
- 不能删除当前登录的自己
- 删除用户不删除其历史月报/预测数据（`user_id` 仍保留在记录中）

## 8. 验收要点

1. 普通用户导航栏只显示被授权页面；直接访问未授权 URL 显示「无权访问」。
2. 直接请求未授权 API 返回 **403**。
3. 用户 A 无法查看/导出/修订用户 B 的月报（404）。
4. 用户 A 无法轮询用户 B 的异步 task_id（404）。
5. 管理员修改权限后，用户刷新页面或重新登录生效。
