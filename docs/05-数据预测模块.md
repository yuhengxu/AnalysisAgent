# 05 · 数据预测模块

> 平台有**两套预测能力**，用途不同，勿混淆：
> - **A. 统计情景模型**（forecast）：纯数值，输出基准/乐观/悲观点位 + 回测。
> - **B. 预测分析表**（prediction skill）：**核心交付物**，LLM 逐项研判影响因素 + 价格区间，导出 Excel。

---

## A. 统计情景模型（ForecastService）

- 服务：`services/forecast.py` ｜ 路由：`api/forecast.py`
- 存表：`forecast_results`

| 方法 | 路径 | 说明 |
|------|------|------|
| `run_forecast(symbol, horizon)` | POST `/forecast/run?symbol=` | 月度均值 → 3 月移动平均+趋势 → 基准；±3 构造乐观/悲观；落库 |
| `list_forecasts(symbol)` | GET `/forecast` | 最近 20 条 |
| `backtest_summary(symbol)` | GET `/forecast/backtest` | MAPE + 方向准确率 |

算法：`baseline = MA3(monthly) + 近3月趋势`，区间 `±3`。属占位级模型，升级见 14-TODO-FORECAST-1（ARIMA/Prophet/ML）。

## B. 预测分析表（PredictionSkill + PredictionService）★核心

### B.1 涉及文件
| 文件 | 职责 |
|------|------|
| `skills/prediction_skill.py` | 采集证据 + 调 LLM 生成结构化预测表 + 降级兜底 |
| `services/prediction.py` | CRUD + 生成编排 + **导出 Excel** |
| `templates/prediction_table.py` | 因素清单/空白骨架/Excel 模板常量 |
| `models/prediction.py` | `predictions` 表 |
| `api/prediction.py` | REST 路由 |
| `skills/sources.py` | 可信数据源（写入 prompt + 标注来源） |

### B.2 数据结构（content_json）
```jsonc
{
  "factors": [
    { "category": "macro", "category_title": "1.宏观及金融因素",
      "id": "1.1", "name": "全球货币政策",
      "importance": 4,            // 1-5
      "judgment": "……（含数据与来源标注）",
      "impact": "促涨"            // 促涨/持平/促跌
    }
    // 共 28 项，覆盖 6 大类
  ],
  "price_forecast": {
    "current_month": { "label": "...", "range_low": 95, "range_high": 100, "avg": 97 },
    "next_month":    { "label": "...", "range_low": 92, "range_high": 97,  "avg": 94 }
  }
}
```

因素清单固定在 `templates/prediction_table.py::PREDICTION_FACTORS`（6 大类共 28 项），与样例 `yuebao/prediction/*.xlsx` 一一对应。

### B.3 生成流程
```
PredictionService.generate(symbol, year, month, provider, model, extra_instruction)
  └─ PredictionSkill.generate()
       ├─ gather_evidence()            # 取上月 Brent/WTI 月度统计、价差、供需、历史因素数
       ├─ if llm.is_enabled():
       │     _generate_with_llm()      # 拼 prompt(因素清单+可信源+证据) → chat_json → _normalize()
       │  else / 异常:
       │     _fallback()               # 用 factor_assessments 历史填充 + 空白骨架
       └─ 返回 {content, evidence, sources_used, llm_used, model}
  └─ 落库 predictions(user_id=当前用户), status=draft
```
- `_normalize()`：强制 importance∈[1,5]、impact∈{促涨,持平,促跌}，保证 28 项齐全且顺序一致。

### B.4 REST 接口
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/prediction/generate` | 生成（body: symbol/year/month/model_provider/model_name/extra_instruction） |
| POST | `/prediction/generate/async` | 异步生成 |
| GET | `/prediction/generate/tasks/{task_id}` | 轮询进度（**校验 task.user_id**） |
| GET | `/prediction` | 列表（**仅当前用户**） |
| GET | `/prediction/{id}` | 详情（含 content/evidence/sources/template） |
| PUT | `/prediction/{id}` | 保存人工校准（content/title），status→reviewed |
| GET | `/prediction/{id}/export` | **导出 Excel**（FileResponse） |
| DELETE | `/prediction/{id}` | 删除（连带删 xlsx） |
| POST | `/prediction/{id}/revise` | **Agent 修订 judgment 字段**（body: factor_idx/field/instruction） |
| GET | `/prediction/sources` | 可信数据源清单 |

### B.4.1 用户隔离

`PredictionService._scope_query` / `get` 严格按 `user_id == 当前用户.id` 过滤，与月报一致。详见 [17-用户管理与权限](./17-用户管理与权限.md)。

### B.5 导出 Excel（还原样例格式）
`PredictionService._build_xlsx()`（openpyxl）：
- 标题、表头、按大类分组着色；
- **重要性 1-5 中被选中的数字加下划线+加粗**；
- **影响三选项中被选中项加下划线+加粗**；
- 形势判断自动换行；末尾价格预测两行 + 三条填表说明 + 全表边框。
- 输出到 `data/exports/prediction_{id}_{时间戳}.xlsx`。
