# 02 · 数据库模块

## 1. 概述

- **数据库**：SQLite，文件位于 `db/energy_platform.db`。
- **ORM**：SQLAlchemy 2.0（`Mapped` / `mapped_column` 风格）。
- **连接与会话**：`backend/app/core/database.py`。
- **表定义**：`backend/app/models/*.py`，统一在 `models/__init__.py` 汇出。

选择 SQLite 的原因：零运维、单文件易迁移、满足当前数据量。**迁移到 PostgreSQL 只需改 `DATABASE_URL`**（连接参数 `check_same_thread` 是 SQLite 专用，迁移时需调整，见 14-TODO）。

## 2. 连接与会话管理（database.py）

```python
engine = create_engine(settings.database_url, connect_args={"check_same_thread": False})
SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)

def get_db():            # FastAPI 依赖注入：每请求一个 Session，结束自动关闭
    db = SessionLocal()
    try: yield db
    finally: db.close()

def init_db():           # 启动时建表（CREATE TABLE IF NOT EXISTS 语义）
    from app import models
    Base.metadata.create_all(bind=engine)
```

- 每个 HTTP 请求通过 `Depends(get_db)` 拿到独立 Session。
- 启动时 `app.main.startup()` 调用 `init_db()` 自动建表；**无独立迁移工具**（变更表结构见 14-TODO 引入 Alembic）。

## 3. 数据表清单（ER 关系）

```
datasets (数据集元信息) ──1:N──┐
                               ├──> price_series      (价格时间序列)
                               ├──> balance_forecasts (机构供需预测)
                               └──> factor_assessments(历史影响因素评估)

users (1) ──N──> reports        (月报实例，user_id 隔离)
users (1) ──N──> predictions    (预测分析表实例，user_id 隔离)

report_templates (1) ──N──> reports
                            forecast_results(统计模型预测结果)
                            agent_runs      (Agent 运行审计)
```

> 注意：当前表之间使用 `dataset_id` 整数列做**逻辑关联**，未声明数据库级外键约束（简化迁移）。`reports.template_id`、`predictions` 同理。强化为外键见 14-TODO。

## 4. 各表字段详解

### 4.1 `datasets` — 数据集元信息
| 字段 | 类型 | 说明 |
|------|------|------|
| id | int PK | 主键 |
| name | str(200) | 数据集名（默认取文件名 stem） |
| source_type | str(50) | `csv` / `xlsx` / `api` |
| category | str(50) | `price` / `balance` / `factor` / `generic` |
| file_path | str(500) | 原始文件落盘路径 |
| schema_json | text | 列结构/sheet 信息 JSON |
| version | int | 版本号（默认 1，预留） |
| row_count | int | 导入行数 |
| status | str(30) | `active` |
| created_at | datetime | 创建时间 |

### 4.2 `price_series` — 价格时间序列（核心事实表）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | int PK | |
| dataset_id | int idx | 来源数据集 |
| symbol | str(50) idx | 品种：Brent/WTI/Dubai/Oman/SC/DTD… |
| trade_date | date idx | 交易日 |
| price | float | 价格 |
| unit | str(30) | 默认 `USD/bbl` |
| source | str(100) | 默认 `CNEEI` |
| created_at | datetime | |

复合索引：`ix_price_symbol_date (symbol, trade_date)` —— 加速按品种+日期范围查询。

### 4.3 `balance_forecasts` — 机构供需预测
| 字段 | 说明 |
|------|------|
| agency | 机构：IEA / EIA / S&P |
| update_date | 机构更新日期（字符串） |
| supply_demand | `供` / `需` |
| period | 周期，如 `2026Q2` |
| value | 数值（百万桶/天） |
| balance_gap | 供需缺口（可空） |

### 4.4 `factor_assessments` — 历史影响因素评估
> 由 `油价预测分析表.xlsx` 导入，作为预测 skill **降级兜底**与口径参考。

| 字段 | 说明 |
|------|------|
| report_month | 形如 `2026-05` |
| category | 大类（宏观/需求/供给/政治/气候） |
| factor_name | 因素名（如 `1.1 全球货币政策`） |
| importance | 重要性 1-5 |
| assessment | 形势判断正文 |
| impact_direction | 促涨 / 持平 / 促跌 |

### 4.5 `forecast_results` — 统计模型预测结果
| 字段 | 说明 |
|------|------|
| symbol / period | 品种 / 周期(YYYY-MM) |
| scenario | baseline / optimistic / pessimistic |
| point_value / low_value / high_value | 点预测 / 区间下限 / 上限 |
| model_name | 如 `ma_trend_v1` |
| evidence_json | 计算依据 JSON |

### 4.6 `report_templates` — 报告模板
| 字段 | 说明 |
|------|------|
| name | 如 `国际油价月报` |
| report_type | `monthly` / `weekly` |
| structure_json | 章节大纲 JSON（见模板模块） |

### 4.7 `reports` — 月报实例
| 字段 | 说明 |
|------|------|
| template_id | 关联模板 |
| title / issue_no / report_date | 标题/期号/报告日期 |
| status | `draft` / `reviewed` |
| **content_json** | **结构化月报内容**（cover/summary/sections/tables/approval），见 07 |
| evidence_json | 证据 + 可信数据源 + 是否用 LLM + charts PNG 路径 |
| docx_path | 最近导出的 Word 路径 |
| **user_id** | 创建者（nullable，admin 可见全部） |
| author/reviewer/approver | 审核信息（冗余，主用 content.approval） |
| created_at / updated_at | |

### 4.8 `predictions` — 预测分析表实例
| 字段 | 说明 |
|------|------|
| title / symbol / year / month | 标题/品种/年/月 |
| status | `draft` / `reviewed` |
| **content_json** | **结构化预测表**（factors[] + price_forecast），见 05 |
| evidence_json | 数据库证据 |
| sources_json | 本次使用的可信数据源 |
| model_name / llm_used | 模型名 / 是否真实调用大模型(0/1) |
| xlsx_path | 最近导出的 Excel 路径 |
| **user_id** | 创建者（nullable，admin 可见全部） |
| created_at / updated_at | |

### 4.9 `users` — 平台用户
| 字段 | 说明 |
|------|------|
| id | 主键 |
| username | 用户名（唯一） |
| hashed_password | bcrypt 哈希 |
| role | `admin` / `user` |
| created_at | 创建时间 |

### 4.10 `agent_runs` — Agent 运行审计
记录每次 `/agent/run` 与 `/agent/chat`：skill、prompt、provider、model、tools_called、evidence、response、charts、duration_ms。用于审计与回溯。

## 5. 关键文件
- `core/database.py`：引擎、会话、`init_db`。
- `models/__init__.py`：**新增表必须在此 import 并加入 `__all__`**，否则不会建表。
- 各 `models/*.py`：单表定义。

## 6. 扩展点
- 新增一张表：建 `models/xxx.py` → 在 `models/__init__.py` 注册 → 重启自动建表。
- 表结构变更：当前靠 `create_all`（**不会自动 ALTER 已存在的表**）。生产应引入 Alembic（见 14-TODO-DB-1）。
